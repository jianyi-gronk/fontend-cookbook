## 1. 搜索引擎录入

- 搜索引擎录入 是 seo 的前提，只有在被收录的基础上才能获得关键词排名

#### 1.1 搜索引擎录入原理

- 搜索引擎通过一个名为 “抓取” 的过程来发现网络上的网页
  - “蜘蛛” 或 “机器人” 是搜索引擎的自动程序，它们会沿着网页之间的链接不断遍历整个互联网，从而发现新网站和更新内容
  - 当蜘蛛发现一个新网页时，它会将页面的 HTML 代码下载下来，这个过程称为 “抓取”
- 随后，搜索引擎会处理并分析抓取到的页面内容，理解其主题和质量，最后将解析后的信息存入一个庞大的数据库，即 “索引”
- 只有被存入 “索引” 的网页，才具备了在搜索结果中出现的资格

#### 1.2 未被录入原因

- 虽然搜索引擎致力于抓取数十亿个网页，但难免也会遗漏部分网站。造成抓取工具遗漏网站的常见原因如下
  - 网站缺乏外部链接
    - 新网站如同孤岛，如果没有其他已被收录的网站链接到它，搜索引擎蜘蛛就很难发现它的存在
    - 但不要付费购买链接，这可能被视为违反搜索引擎的垃圾内容政策
  - 网站刚刚上线
    - 搜索引擎发现和抓取新网站需要一定的时间，对于全新网站或现有网站的重大更新，这个过程可能需要几周
  - 网站设计存在技术障碍
    - 如果网站是基于其他一些专业技术（ 而非 HTML ）构建的，可能导致无法有效抓取内容
    - 因此如果网站主要依赖图片或视频来传递信息，而缺乏可供抓取的文字内容，搜索引擎也难以理解网站主题
  - 网站存在 抓取错误 或 访问限制
    - 抓取错误：蜘蛛在尝试抓取时可能遇到服务器错误（ 如 5xx 错误 ）、客户端错误（ 如 4xx 错误 ）等
    - 访问限制：网站可能因 robots.txt 文件设置不当、需要登录或 IP 屏蔽等原因，无意中禁止了搜索引擎的访问，同时确保可在 无痕式窗口 中访问网站
  - 偶然被遗漏
    - 尽管搜索引擎能抓取海量网页，但仍有可能偶然漏掉一些规模较小的新网站

#### 1.3 检测是否被录入

- `site:` 搜索运算符
  - `site:` 查询是一个搜索运算符，可以使用它请求来自运算符中指定的特定网域、网址或网址前缀的搜索结果
- 示例
  - `site:example.com`
    - 返回已编入索引并呈现给用户的网址的列表，仅显示来自 `example.com` 网域（ `www.example.com` 和 `recipes.example.com` ）的结果
  - `site:https://www.example.com/ramen关键词`
    - 显示包含以 `https://www.example.com/ramen` 开头的网址并与 关键词 相关的网页的结果
  - `site:https://example.com/recipes/tsukemen.html`
    - 可帮助了解特定网址是否已编入索引并呈现给用户
  - `site:example.com viagra casino`
    - 有助于发现和监控网站上的垃圾内容问题
  - `site:https://example.com/关键词`
    - 显示查询 关键词 会显示网站上的哪些网址
  - `site:https://example.com/recipes/tsukemen.html关键词`
    - 显示是否已按照 关键词 将特定网址编入索引
- 细节
  - 如果不含查询的关键词，例如 `site:example.com`
    - 系统不会对结果进行排名
    - 系统通常会在顶部显示带有该前缀的最短网址，但除此之外则会相对随机地显示结果
  - 是否返回所有网站
    - 运算符不一定会返回按照查询中指定的前缀编入索引的所有网址
    - 所以如果想确定有多少网址已按照某个前缀编入索引并呈现给用户，`site:example.com` 不太适用
- 扩展
  - `src:` 搜索运算符
    - 例如：`src: https://example.com/media/carrot.jpg`
    - `src:` 搜索运算符将返回在 `src` 属性中引用了运算符中提供的图片网址的网页
    - 该运算符会显示来自任何网域的网页，而不仅仅是来自运算符中指定的网址所在网域的网页。这可能有助于了解您在网站上托管的哪些图片被其他网站建立了热链接
  - `imagesize:` 搜索运算符
    - 例如：`imagesize: 1500x1000`
    - `imagesize:` 搜索运算符将显示运算符中指定的尺寸的图片，必须以 “宽 x 高” 的格式指定尺寸
    - 此运算符与 `src:` 和 `site:` 运算符配合使用时会非常有帮助
      - 可以查找网站上已编入索引的特定大小图片，例如 `src: https://example.com/media/carrot.jpg imagesize: 500x1200`
      - 可以按照精确的大小查找图片，例如 `site: https://example.com/ imagesize: 500x1200`

#### 1.4 加速搜索引擎录入

- 使用官方站长工具（ 最快、最直接的方法 ）
  - 这是加速收录最重要的一步，它为您提供了与搜索引擎直接沟通的官方渠道
  - 站长工具使用前必须注册并验证目标网站的所有权，例如 Goole 的 Google Search Console，Bing 的 Bing Webmaster Tools，百度的 百度搜索资源平台
  - 关键操作
    - 提交网站地图（ Sitemap ）
      - 在站长工具中提交您的 sitemap.xml 文件
      - 这个文件就像网站的 “目录”，清晰地列出了所有重要页面的网址及其更新频率，极大方便了搜索引擎蜘蛛的系统性抓取
    - 提交网址
      - 对于最重要的页面（ 如首页、核心产品页 ），可以使用站长工具提供的 “网址检查” 工具，直接提交单个 URL 请求索引
    - 监控抓取信息
      - 在站长工具中，可以查看蜘蛛抓取您网站时是否遇到了错误（ 如 404、服务器错误等 ），并及时修复，确保抓取渠道畅通
- 获取高质量外部链接（ 自然引导 ）
  - 搜索引擎蜘蛛沿着链接在互联网上 “爬行”
  - 原理：从其他已被收录且受信任的网站指向新网站的链接，就像是为蜘蛛搭建了一座通往新网站的 “桥梁”
  - 方法
    - 在相关的行业论坛、博客评论区（ 需有价值，非垃圾评论 ）中留下网站的链接
    - 在社交媒体（ 如 LinkedIn, Twitter, 知乎，豆瓣小组等 ）上分享内容
    - 与其他站长进行友情链接交换
  - 注意：质量远大于数量，一个来自权威网站的链接，胜过几十个低质量链接
- 优化内部链接结构（ 内部引导 ）
  - 确保网站内部的链接网络清晰、通畅，让蜘蛛能够轻松遍历所有页面
  - 创建清晰的导航菜单和面包屑导航
  - 在文章内容中，通过上下文锚文本自然地链接到网站内的其他相关页面
  - 建立一个网站地图页面（ HTML Sitemap ），方便用户和蜘蛛一览全站结构
- 创建并优化 robots.txt 文件
  - 作用：这个文件位于网站的根目录（ 如 `yourdomain.com/robots.txt` ），用于告诉搜索引擎蜘蛛哪些目录或文件可以或不可以抓取
  - 注意不要错误地使用 `Disallow:` 指令屏蔽了希望被收录的重要资源（ 如 CSS、JS 文件或内容页面 ），否则会严重影响蜘蛛的理解和收录

## 2. SEO 原理

#### 2.1 抓取与渲染

- 即 “读取” 页面内容
- 抓取
  - 搜索引擎的 “蜘蛛” 或 “机器人” 会沿着链接遍历互联网，发现网页，并将其 HTML 代码下载下来
- 渲染
  - 因为现代网页大量使用 JavaScript 来动态生成内容，所以搜索引擎不会只看服务器返回的初始 HTML 代码
  - 常见搜索引擎（ Bing，Google，Baidu 等 ）会像一个浏览器一样去渲染整个页面，执行 JavaScript，从而看到最终用户能看到的所有文本、图片和链接
  - 不同搜索引擎的渲染能力和速度存在差异
    - 通常，Google 的渲染能力最强、速度最快；Bing 也在持续跟进；而 百度 虽然已具备 JS 渲染能力，但其处理速度和深度可能相对滞后，因此百度对 JS 网站的收录仍可能存在延迟
    - Bing 通过 Bingbot（ 网络抓取机器人 ）建立一个可搜索的索引，而 Bingbot 已采用基于 Chromium 的 Edge 引擎渲染网页（ 与 Google 相同 ），这样网站开发人员对于 seo 兼容性的考虑会少一点
- “读取” 时，页面状态为未登录
  - 搜索引擎在绝大多数情况下，都是以未登录的匿名用户身份来抓取和渲染页面的
  - 就像是一个第一次访问网站、没有携带任何登录 Cookie 或会话信息的全新浏览器
  - 因此搜索引擎无法索引登录后内容，也不会将它们纳入排名考量，因为它们无法代表普通匿名用户的访问体验

#### 2.2 处理与理解

- 即 “理解” 页面内容
- 拿到所有文本后，搜索引擎并不是简单地把它存起来，它会进行复杂的自然语言处理来分析内容
  - 关键词分析
    - 识别内容的核心主题和关键词
    - 现在的关键词匹配不再是简单的 “出现次数”，而是更注重语义相关性和上下文
  - 实体识别
    - 识别出文本中的人物、地点、事件、产品等具体对象
  - 主题建模
    - 理解整个页面或段落是关于什么主题的
  - 内容质量评估
    - 分析内容的独特性、深度、可读性、是否满足用户搜索意图等

#### 2.3 排序

- 即决定返回结果的排名
- 在理解了页面内容之后，搜索引擎会使用数百个排名因素来综合计算每个页面的得分，这些信号可以大致分为几类：内容相关因素，页面技术因素，站外权威性与信任度因素，用户行为信号
- 内容相关因素
  - 关键词与语义
    - 内容是否与搜索查询高度相关？是否全面覆盖了主题？
  - 内容质量与深度
    - 是浅薄的碎片化内容，还是有深度的权威指南？
  - 新鲜度
    - 内容是否最新？对于某些话题（ 如新闻、科技 ），时效性非常重要，但搜索 “莎士比亚生平” 则几乎不受时间影响
  - 用户体验
    - 内容是否易于阅读？排版是否清晰？是否有图片、视频等多元素？
- 页面技术因素
  - 标题标签 和 Meta 描述
    - 虽然不再是直接排名因素，但对点击率有巨大影响
  - URL 结构
    - 清晰、包含关键词的 URL 更好
  - 内部链接
    - 网站内部其他页面是否链接到此页？这有助于传递权重和建立网站结构
  - 移动端友好性
    - 页面在手机上的加载速度和显示效果
  - 核心网页指标
    - 页面加载速度、交互性 和 视觉稳定性 等
- 站外权威性与信任度因素
  - 反向链接
    - 非常重要的因素，如果其他高质量网站链接到你的页面，相当于投了 “信任票”
    - 链接的数量和质量是衡量页面权威性的关键指标
  - 品牌信号
    - 用户是否搜索你的品牌名？你的网站在社交媒体上的提及度如何？
- 用户行为信号
  - 点击率
    - 在搜索结果中，用户是否更愿意点击你的链接？
  - 停留时间
    - 用户点击后，是否在页面上停留了较长时间？（ 表明内容有价值 ）
  - 跳出率
    - 用户是否立刻返回搜索结果？（ 可能表明内容不相关或质量差 ）

## 3. SEO 评估指标

- 评估 SEO 收益需要监控三类指标，即 过程指标 → 目标指标 → 业务指标

#### 3.1 过程指标

- 过程指标 用于 监测和优化 SEO 工作本身的有效性，它们反映了搜索引擎与网站互动的基础健康度，是达成最终目标的前提
- 反向链接数量与质量
  - 指向你网站的外部链接的数量，以及这些链接来源网站本身的权威性
  - 高质量的反向链接仍然是搜索引擎最信任的“投票”机制，是提升域名权威性和排名的核心驱动力
  - 通过 Google Search Console 的 “链接” 报告，或第三方外链分析工具（ 如 Ahrefs, Majestic ）
- 引用域数量
  - 有多少个独立的域名链接到你的网站
  - 自 100 个不同域名的各 1 个链接，通常比来自 1 个域名的 100 个链接更有价值，它代表了更广泛的认可
- 核心网页指标
  - 由 Google 提出的一组量化用户体验的指标，主要包括
    - LCP：衡量加载速度。理想时间应在 2.5 秒以内
    - FID：衡量交互性。理想时间应在 100 毫秒以内
    - CLS：衡量视觉稳定性。理想分数应低于 0.1
  - 通过 Google Search Console 的 核心网页指标 报告、PageSpeed Insights 查看
  - 它们是 Google 的直接排名因素，并且严重影响用户的停留与转化
- 已编入索引页面数
  - 网站有多少个页面被搜索引擎存入索引库
  - 这是排名的前提，如果页面未被收录，就不可能获得流量
  - 通过  `site:yourdomain.com`  指令进行粗略估算，或在 Google Search Console 的 “索引” 版块查看精确数字
- 收录率
  - （ 已索引页面数 / 网站总页面数 ）x 100%
  - 高收录率说明网站结构良好，所有重要页面都被发现；低收录率则表明可能存在重复内容、低质量页面或技术问题导致页面无法被收录
- 爬虫抓取统计
  - 搜索引擎蜘蛛访问你网站的频率、抓取的页面数以及抓取时是否遇到错误。
  - 抓取频率高通常意味着网站更新频繁、权威性高；抓取错误则需要及时修复，以免影响收录
  - 通过 Google Search Console 中的 “抓取统计信息” 和 “爬虫抓取” 部分查看

#### 3.2 目标指标

- 目标指标 是 SEO 工作带来的直接成果，它们清晰地展示了网站在搜索引擎中的能见度和吸引力
- 关键词排名位置
  - 目标关键词在搜索结果中的平均排名
  - 排名越高，通常获得的点击和流量就越多，排名趋势可以衡量 SEO 策略是否有效
  - 通过 Google Search Console 的 “搜索效果” 报告，或第三方 SEO 工具（ 如 SERPROBOT，Small SEO Tools 等 ）查看
- 搜索展现份额
  - 网站在所有包含你关键词的搜索结果中，实际被用户看到的展示次数所占的百分比
  - 即使排名很高，也可能因为竞争对手的富媒体摘要（ 如精选片段、知识图谱 ）占据了大部分展示空间，这个指标帮助了解真正的可见度
  - 通过 Google Search Console 的 “搜索效果” 报告中，对比 “展示次数” 和 “搜索展现份额” 分析
- 点击率
  - 即搜索结果被展示后，用户点击它的比例，（ 点击次数 / 展示次数 ）x 100%
  - 高点击率意味着 标题标签 和 元描述 足够吸引人，优化这两个元素可以有效提升 CTR，即使排名不变也能带来更多流量
  - 通过 Google Search Console 的“搜索效果” 报告查看
- 自然搜索流量
  - 通过搜索引擎的自然搜索结果（ 非广告 ）访问你网站的用户数量和会话次数
  - 通过 Google Analytics 等网站分析工具
  - 这是 SEO 工作的核心成果，直接反映了 SEO 带来的价值
- 平均停留时长
  - 用户通过搜索结果进入你的网站后，平均每次会话在网站上停留的时间
  - 较长的停留时间通常暗示内容质量高，符合用户搜索意图，搜索引擎会将其视为积极信号
  - 通过 Google Analytics 查看
- 跳出率
  - 仅浏览了一个页面便离开网站的访问次数占总访问次数的百分比
  - 高跳出率可能意味着页面内容不相关、用户体验差或未能满足用户需求。但需结合上下文分析，例如博客文章的跳出率天然较高
  - 通过 Google Analytics 查看

#### 3.3 业务指标

- 业务指标 用于衡量 SEO 工作带来的最终业务收益
- 转化数
  - 由自然搜索流量直接产生的潜在客户数量、订单数量 或 营业收入
  - 代表了流量带来的总收益，转换数高代表 SEO 的收益大
- 转化率
  - 通过自然搜索流量完成目标动作（ 如 购买、注册、下载 ）的用户百分比
  - 代表了流量的质量，高转化率意味着 SEO 吸引了精准的目标用户，并满足了他们的需求
